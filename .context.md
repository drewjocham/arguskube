## Overview

KubeWatcher is a native macOS SRE console built with **Wails v2** (Go backend + Vue 3 frontend). It provides a single-pane-of-glass view into Kubernetes clusters with real-time monitoring, workload management, incident response, AI diagnostics, and automation workflows.

**Module:** `github.com/argues/kube-watcher`

## Architecture

### Dual-Mode Runtime

The app runs in two modes simultaneously:

1. **Wails Desktop** — Vue frontend calls Go methods via `window.go.pkg.App.MethodName()`. Direct, zero-latency, no HTTP.
2. **SaaS HTTP** — Same methods exposed as `POST /api/{MethodName}` on port 8080 via a reflection-based dispatcher in `api/pkg/server.go`.

The frontend `callGo()` bridge in `useBridge.js` auto-detects which mode is available and falls back transparently.

## Known Technical Debt

### Critical
- [ ] **App god object** — `api/pkg/app.go` has 63 methods and 17 dependencies. Needs decomposition into focused service facades.
- [ ] **Reflection HTTP dispatcher** — `api/pkg/server.go` uses `reflect.ValueOf().MethodByName()`. Replace with Connect-Go codegen.

### High
- [ ] **No retry/rate-limit on AI client** — `internal/ai/deepseek.go` makes bare HTTP calls.
- [ ] **Incidents store race condition** — `persist()` can lose data under concurrent writes.
- [ ] **Workflow store non-atomic writes** — Partial write corrupts JSON file.

### Medium
- [ ] Copy-paste CRUD patterns across 4 domains (extract generic)
- [ ] Small touch targets on destructive actions (IncidentLog, S3Notebook)
- [ ] LogExplorer has redundant sidebar + filter bar
- [ ] Go test compilation not verified in CI (Go not in sandbox)
