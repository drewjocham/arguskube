## Overview

Argus is an SRE console for Kubernetes operations. It runs in three modes:

| Mode | Entry Point | Target |
|------|-------------|--------|
| **Desktop** | `kube/backend/main.go` (Wails v2) | Native macOS app |
| **SaaS** | `kube/backend/cmd/server/main.go` (HTTP, headless) | K8s deployment / Docker |
| **Agent** | `agent/cmd/agent/main.go` (DaemonSet) | In-cluster per node |

Additional binaries: `kube/alert-ingress/` (webhook→PubSub), `kube/mcp/` (MCP AI tools).

**Module:** `github.com/argues/argus` | **Go:** 1.25.0 | **Wails:** v2.12.0

## Architecture

### Dual-Mode Runtime (Desktop)

1. **Wails Desktop** — Vue frontend calls Go via `window.go.pkg.App.MethodName()`. No HTTP.
2. **SaaS HTTP** — Same methods exposed as `POST /api/{MethodName}` on port 8080 via reflection dispatcher in `server.go`.

The frontend `callGo()` bridge auto-detects mode and falls back transparently.

### Multi-Process Architecture

```
Desktop App (Wails) / SaaS Server (headless)
  ├── K8s API (direct client-go)
  ├── In-Cluster Agent (port-forward / WebSocket tunnel)
  ├── Argo CD REST API
  ├── DeepSeek / Ollama API
  ├── Anomstack (anomaly detection)
  ├── S3 (notebooks)
  └── SQLite (incidents, workflows, anomaly settings)
```

## Decision Log

| Date | Decision | Rationale |
|------|----------|-----------|
| 2026-04 | Wails v2 over Electron | Native binary, no Chromium overhead, Go backend |
| 2026-04 | SQLite over JSON files for persistence | Concurrent-safe, no partial writes, migration-based schema |
| 2026-04 | `modernc.org/sqlite` (pure Go) | No CGo requirement, single binary |
| 2026-04 | Reflection dispatcher for HTTP API | Quick path to dual-mode; acknowledged tech debt |
| 2026-04 | DeepSeek over OpenAI | Cost efficiency for diagnostic agent |
| 2026-05 | Module rename to `argues/kube-watcher` | Organization rebrand |
| 2026-05 | Split `useWails.js` into domain composables | 14 domain files + barrel file, all tests passing |
| 2026-05 | Go 1.25.0 baseline | Wails v2.12 requires 1.23+, chosen for generics ergonomics |
| 2026-05 | GitHub Actions CI pipeline | Build/test/lint on push/PR for all Go modules + Vue |
| 2026-05 | mTLS for agent WebSocket tunnel | `internal/tlsconfig/certs.go` — CA-signed certs, TLS 1.3 |
| 2026-05 | Two `go.mod` files (backend/ + agent/) | Separate dependency trees, avoid bloating agent binary |
| 2026-05 | Agent port-forward via `client-go` SPDY | No separate tunnel port needed, uses existing kubeconfig |
| 2026-05 | DeepSeek client with retry/backoff | 3 retries, exponential backoff, Retry-After header support |

## Known Technical Debt

### Critical
- [ ] **App god object** — `api/pkg/app.go` has 17 dependencies injected as fields. Needs service facades.
- [ ] **Reflection HTTP dispatcher** — `api/pkg/server.go` uses `reflect.ValueOf().MethodByName()`. Replace with Connect-Go codegen.
- [ ] **Desktop-SaaS conflation** — `api/pkg/` mixes Wails bindings with SaaS HTTP server. Separate packages needed.
- [ ] **No SaaS auth middleware** — `server.go` has zero auth on `/api/*`. All endpoints public.
- [ ] **No authentication implemented** — `auth-architecture.md` defines interfaces, no implementation wired yet.

### High
- [ ] **Hub bottleneck** — `api/pkg/hub.go` uses `map[string]*AgentConnection + sync.RWMutex`. Won't scale past ~50 agents.
- [ ] **Agent not wired to DeepSeek** — The "AI-powered" label is misleading. Agent context assembly (`internal/context/diagnose.go`) is a rules engine. DeepSeek not connected for diagnosis.
- [ ] **Workflows engine** — Custom DAG executor. REVIEW_SUMMARY recommends replacing with Temporal or Argo Workflows.
- [ ] **Incident store race condition** — `persist()` can lose data under concurrent writes (though now SQLite-backed, review needed).
- [ ] **Workflow store non-atomic writes** — Partial write corrupts JSON file (SQLite migration partially addressed).
- [ ] **No tenant isolation model** — SaaS has no multi-tenant design, no API gateway, no message broker.

### Medium
- [ ] Copy-paste CRUD patterns across 4 domains
- [ ] Small touch targets on destructive actions
- [ ] LogExplorer has redundant sidebar + filter bar
- [ ] `go-chi/chi/v5` and `dgraph-io/badger/v4` in `go.mod` — likely unused dead deps
- [ ] `agent/` Go module has zero tests
- [ ] Event loop continues polling when Wails window closed (no pause-on-hide)
- [ ] `internal/k8s/resources.go` at 1817 lines handles 17 resource types — violates SRP
- [ ] No mockery config (`.mockery.yaml` missing)
- [ ] No Playwright E2E tests (TEST_PLAN.md P3 not started)
- [ ] No upgrade CTA when Pro features blocked (silent failure)
